Dave's HTML Style Guide Cheatsheet

Rule Description Example
Declare doc type Always declare the document type as the first line in your document. <!DOCTYPE html>
Lowercase element names Although HTML allows either case, lowercase is standard <body> NOT <BODY>
Always close elements HTML doesn't require closing tags to work, but it can lead to unexpected results (except void elements! eg. <br>, <col>, <img>, <input>, <link>, <source>, <track>) <section>
  <p>Some content</p>
</section>
Lowercase attribute names As with element names. <a href="https://www.anadress.com">
Quote attribute values attributes work without quotes (unless there are spaces) but are easier to read with. <table class="striped">
Always specify alt, width, and height for images If an image can't be displayed it is important to have an alt message. Width and height improves display by allowing the browser to reserve the space for the image. <img src="animage.png" alt="Missing image" style="width:128px;height:128px">
Space-less equal signs Although spaces are allowed it is easier to read without. <link rel="stylesheet" href="styles.css">
Avoid long code lines It is easier not to have to scroll an HTML editor
Blank lines and indentation Only add blank lines to separate large or logical code blocks. Use two or four spaces of indentation (pick one and stay consistent), not the tab key.
Never skip <title> It is required in HTML. It is additionally very important for SEO. Make it as accurate and meaningful as possible. <title>HTML Style Guide and Conventions</title>
Never skip <html> or <body> Ommitting body can produce errors in older browsers or crash some software. <html>
<body>
Don't skip <head> It works without but don't. <head>
Close empty elements To help with XML/XHTML software add the closing tags to the void tags. <meta charset="utf-8" />
add the lang attribute The language inside the html tag allows search engines and browsers to detect the language of the page. <html lang="en-us">
Define metadata For the same reason, include the character encoding asap in the head of a document. <head>
  <meta charset="UTF-8">
...;
Set the viewport This meta element allows browsers to fit the page's dimensions and scaling to whatever device it's on. <meta name="viewport" content="width=device=width, initial-scale=1.0">
Comments Short comments on one line, long ones indented on multiple lines. <!-- Comment 1 is short -->
<!--
  Comment 2 is looooooooooo
  oooooooooooooooooooooong.
-->
Lowercase file names Some web servers are case sensitive, others are not. Moving from one to the other can break sites if file names are inconsistent. <glasgow.jpg NOT Glasgow.jpg>
file extensions Files should have the correct extensions. index.html
style.css
myscript.js
